## 3. 表设计

### 3.1 student 表（学生表）

作用：保存学生的基础信息。

字段设计：

| 字段名 | 类型 | 含义 |
|--------|------|------|
| id | INT | 学生ID，主键 |
| name | VARCHAR(50) | 学生姓名 |
| age | INT | 学生年龄 |

建表 SQL：

```sql
CREATE TABLE student (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50),
    age INT
);
````

---

### 3.2 course 表（课程表）

作用：保存课程信息。

字段设计：

| 字段名         | 类型          | 含义      |
| ----------- | ----------- | ------- |
| id          | INT         | 课程ID，主键 |
| course_name | VARCHAR(50) | 课程名称    |

建表 SQL：

```sql
CREATE TABLE course (
    id INT PRIMARY KEY AUTO_INCREMENT,
    course_name VARCHAR(50)
);
```

---

### 3.3 score 表（成绩表）

作用：保存学生和课程之间的成绩信息。

字段设计：

| 字段名        | 类型           | 含义                |
| ---------- | ------------ | ----------------- |
| id         | INT          | 成绩记录ID，主键         |
| student_id | INT          | 学生ID，关联 student 表 |
| course_id  | INT          | 课程ID，关联 course 表  |
| score      | DECIMAL(5,2) | 分数                |

建表 SQL：

```sql
CREATE TABLE score (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_id INT,
    course_id INT,
    score DECIMAL(5,2)
);
```

## 4. 三张表之间的关系

### 4.1 student 和 course 的关系

一个学生可以选多门课，一门课也可以被多个学生选择，因此 student 和 course 是 **多对多关系**。

### 4.2 为什么需要 score 表

因为 student 和 course 是多对多关系，不能直接在 student 表里写课程，也不能直接在 course 表里写学生，所以需要单独设计一张 score 表来保存关系。

score 表中：

* `student_id` 表示是哪一个学生
* `course_id` 表示是哪一门课程
* `score` 表示这个学生这门课的成绩

也就是说，score 表既保存了“学生和课程的关系”，也保存了成绩数据。

---

## 5. 为什么这样设计

### 5.1 避免数据冗余

如果直接把学生姓名、课程名、成绩都写在一张表里，例如：

| 学生名 | 课程名   | 分数 |
| --- | ----- | -- |
| 张三  | Java  | 95 |
| 张三  | MySQL | 90 |

会出现问题：

* 学生姓名重复
* 课程名称重复
* 修改学生姓名时需要改多条数据
* 数据不一致风险更大

所以应该拆分成 student、course、score 三张表。

---

### 5.2 便于维护

这样设计后：

* 学生改名，只需要改 student 表
* 课程改名，只需要改 course 表
* 成绩只保存在 score 表

数据库结构更清晰，后续扩展也更方便。

## 6. 对三范式的初步理解

### 第一范式（1NF）

要求：字段不能再拆分，必须是最小单位。

例如：

* `name` 是一个字段
* `age` 是一个字段
* `course_name` 是一个字段

这些字段都是原子性的，因此满足第一范式。

---

### 第二范式（2NF）

要求：每条记录都依赖主键，不能只依赖主键的一部分。

当前每张表都有主键 `id`，其他字段都依赖这个主键，因此基本满足第二范式。

---

### 第三范式（3NF）

要求：非主键字段不能依赖其他非主键字段，尽量减少冗余。

当前设计中：

* student 表只保存学生信息
* course 表只保存课程信息
* score 表只保存关联和成绩

没有把学生名、课程名重复写到 score 表中，因此减少了冗余，基本满足第三范式。

## 7. 今天练习过的基础 SQL

### 7.1 查询所有学生

```sql
SELECT * FROM student;
```

### 7.2 查询学生姓名

```sql
SELECT name FROM student;
```

### 7.3 条件查询

```sql
SELECT * FROM student WHERE age > 20;
```

### 7.4 插入数据

```sql
INSERT INTO student (name, age) VALUES ('张三', 20);
```

### 7.5 统计人数

```sql
SELECT COUNT(*) FROM student;
```

### 7.6 多表查询

```sql
SELECT student.name, course.course_name, score.score
FROM score
JOIN student ON score.student_id = student.id
JOIN course ON score.course_id = course.id;
```
