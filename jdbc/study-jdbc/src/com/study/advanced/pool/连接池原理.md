# 连接池原理.md

## 1. 什么是连接池

连接池（Connection Pool）本质上是一个“提前创建好多个数据库连接并重复使用”的技术。

程序需要操作数据库时，不再每次都重新创建 Connection，而是直接从连接池中获取一个可用连接；使用完后，也不是直接销毁，而是归还到连接池中，供下次继续使用。

---

## 2. 为什么要用连接池

如果不使用连接池，传统 JDBC 操作流程通常是：

1. 创建数据库连接
2. 执行 SQL
3. 关闭连接

每次执行一次数据库操作，都要重复创建和销毁连接。

这个过程开销较大，因为数据库连接的创建本身比较耗时，涉及：

- 网络通信
- 权限认证
- 数据库资源分配

如果频繁执行 SQL，就会导致性能下降。

---

## 3. 连接池解决了什么问题

连接池的作用主要有两个：

### 3.1 提高性能

连接提前创建好，后续直接复用，减少重复创建连接的耗时。

### 3.2 提高系统稳定性

连接池会统一管理连接数量，避免因为频繁创建连接导致数据库压力过大。

---

## 4. 连接池工作流程

连接池的大致工作过程如下：

1. 程序启动时，连接池先创建多个数据库连接
2. 程序需要连接时，从池中取出一个连接
3. 程序执行 SQL
4. 程序使用完后，将连接归还到连接池
5. 下次再次复用该连接

可以理解为：

- 不使用连接池：每次打车
- 使用连接池：公司班车循环使用

---

## 5. 常见连接池

Java 开发中常见的连接池有：

- Druid
- HikariCP
- C3P0（较老）
- DBCP

其中：

- **HikariCP**：性能高，Spring Boot 默认连接池
- **Druid**：功能丰富，监控能力强，学习阶段常见
