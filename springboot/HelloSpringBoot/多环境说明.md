# 一、多环境配置的核心作用
    多环境配置（dev/prod）的核心目的是：一套代码，适配不同使用场景，避免频繁修改配置、减少开发与上线的冲突，提升开发效率和项目稳定性。
    结合本项目（HelloSpringBoot），具体作用如下：
    - 开发环境（dev）：供自己本地开发、测试使用，配置本地数据库、调试端口（8080），方便开发时调试接口、修改代码。
    -生产环境（prod）：项目上线部署到服务器时使用，配置服务器相关参数（如端口80，正式数据库账号密码），保证项目在线上稳定运行。
    - 一键切换：通过主配置文件的spring.profiles.active，无需修改其他配置，即可切换开发/生产环境，避免手动修改配置出错。
# 二、配置文件目录结构
    所有配置文件均存放于项目 src/main/resources 目录下，目录结构如下（与本项目完全对应）：
    src/main/resources/
    ├─ application.yml        # 主配置文件（全局公共配置）
    ├─ application-dev.yml    # 开发环境配置（本地使用）
    └─ application-prod.yml   # 生产环境配置（服务器使用）
# 三、各配置文件详细说明
    1. 主配置文件：application.yml
    核心作用：定义项目全局公共配置、指定当前激活的环境，是多环境配置的入口。
    spring:
        application:
            name: HelloSpringBoot  # 项目名称（全局公共配置，所有环境共用）
        profiles:
            active: dev            # 激活的环境（dev=开发环境，prod=生产环境）
        说明：修改 active 的值（dev/prod），即可一键切换环境，无需修改其他配置文件。
    2. 开发环境配置：application-dev.yml
    核心作用：配置本地开发所需参数，适配本地电脑的运行环境。
    spring:
        datasource:
        url: jdbc:mysql://localhost:3306/study_sql?useSSL=false&serverTimezone=UTC  # 本地数据库地址
        username: root                                                              # 本地数据库账号
        password: 123456                                                           # 本地数据库密码

    server:
    port: 8080  # 本地开发端口（8080为常用调试端口，避免与其他程序冲突）
    3. 生产环境配置：application-prod.yml
    核心作用：配置服务器运行所需参数，适配线上部署环境。
    spring:
        datasource:
            url: jdbc:mysql://localhost:3306/study_sql?useSSL=false&serverTimezone=UTC  # 服务器数据库地址（后续可替换为服务器IP）
            username: root                                                              # 服务器数据库账号（线上建议使用专用账号，提升安全性）
            password: 123456                                                           # 服务器数据库密码（线上需修改为复杂密码）

    server:
    port: 80  # 线上服务器常用端口（80端口无需输入端口号即可访问，如：http://xxx.xxx.xxx.xxx）
# 四、配置加载顺序
## SpringBoot 加载多环境配置的顺序固定，优先级从高到低如下：
    1. 先加载主配置文件 application.yml（全局公共配置）；
    2. 再加载当前激活环境的配置文件（如激活dev，则加载 application-dev.yml）；
    3. 若主配置与环境配置有同名配置，环境配置的内容会覆盖主配置（例如：主配置和dev配置都配置了server.port，以dev的8080为准）。
# 五、使用注意事项
    - yml 配置文件严格要求：冒号后面必须加空格、缩进使用2个空格（不能用Tab），否则配置会失效。
    - 开发环境与生产环境的数据库、端口等参数尽量区分开（本示例暂用相同数据库，上线时需替换为服务器数据库信息）。
    - 建议删除旧的 application.properties 配置文件，避免两种配置文件共存导致冲突、配置失效。
    - 切换环境后，需重启SpringBoot项目，配置才能生效。
# 六、环境切换示例
## 1. 切换到开发环境（默认）：
   spring:
   profiles:
   active: dev
   重启项目后，项目使用8080端口、本地数据库，适合本地开发调试。
## 2. 切换到生产环境：
   spring:
   profiles:
   active: prod
   重启项目后，项目使用80端口、服务器数据库，适合上线部署。
