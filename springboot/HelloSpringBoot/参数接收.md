# 一、三大参数注解区别速览
    注解	            作用	                                适用场景	                    请求方式	        数据格式
    @PathVariable	路径变量，获取 URL 路径中的占位符参数	    RESTful 风格路径传参、资源定位	GET/DELETE/PUT	拼接在 URL 路径中
    @RequestParam	请求参数，获取地址栏？后的参数、表单参数	普通键值对传参、传统表单提交	    GET/POST	    key=value 键值对
    @RequestBody	请求体参数，读取请求体 JSON/XML 数据	    复杂对象、JSON 提交、批量传参	POST/PUT	    JSON / XML
# 二、@PathVariable 路径变量
1. 核心作用
   绑定 URL 路径中的动态占位符，是 RESTful 风格核心传参方式。
2. 基础语法
```java
   // 路径：/user/1001
   @GetMapping("/user/{id}")
   public Result getUser(@PathVariable Integer id){
   // id = 1001
   }
  ```
3. 多参数接收
```java
// 路径：/user/1001/zhangsan
@GetMapping("/user/{id}/{name}")
public Result getUser(
    @PathVariable Integer id,
    @PathVariable String name
){}
```
4. 别名绑定（参数名不一致）
```java
   @GetMapping("/user/{uid}")
   public Result getUser(@PathVariable("uid") Integer userId){}
```
5. 特点
   参数必传，不传直接 404
   常用于：查询单个资源、删除、修改（/user/1）
# 三、@RequestParam 请求参数
1. 核心作用
   获取 URL 拼接参数（?key=value）、form 表单键值对。
2. 基础使用
```java
   // 请求地址：/list?page=1&size=10
   @GetMapping("/list")
   public Result list(
   @RequestParam Integer page,
   @RequestParam Integer size
   ){}
```
3. 常用属性
   value / name：绑定请求参数名
   required：是否必传，默认 true
   defaultValue：参数默认值
```java
   @GetMapping("/list")
   public Result list(
   @RequestParam(value = "page", required = false, defaultValue = "1") Integer page
   ){}
```
4. 接收数组 / 集合
```java
   // /ids?ids=1,2,3
   @GetMapping("/ids")
   public Result getIds(@RequestParam List<Integer> ids){}
```
5. 特点
   传统开发常用，非 RESTful 标准
   适合简单少量键值对参数
# 四、@RequestBody 请求体
1. 核心作用
   接收请求体中的 JSON 格式数据，自动封装为实体类。
2. 使用前提
   请求头：Content-Type: application/json
   只支持：POST、PUT 等携带请求体的请求
3. 接收实体对象
```java
   // 前端传 JSON
   /*
   {
   "username": "张三",
   "age": 20
   }
   */
   @PostMapping("/user/add")
   public Result addUser(@RequestBody User user){}
```
4. 接收 Map 通用参数
```java
   @PostMapping("/data")
   public Result getData(@RequestBody Map<String,Object> map){}
```
5. 注意事项
   一个接口只能有一个 @RequestBody
   不能接收 form-data、x-www-form-urlencoded 表单
   前端必须传 JSON，否则报错
# 五、RESTful 接口设计规范
   1. 核心思想
      用请求方式表示操作行为，用URL 路径表示资源。
      表格
      请求方式	操作	示例
      GET	查询资源	GET /user/{id} 查询单个用户
      POST	新增资源	POST /user 新增用户
      PUT	全量修改资源	PUT /user/{id} 修改用户
      DELETE	删除资源	DELETE /user/{id} 删除用户
   2. RESTful 标准接口示例
```java
    // 1. 根据ID查询（查）
    @GetMapping("/user/{id}")
    public Result getById(@PathVariable Integer id){}

    // 2. 新增用户（增）   
    @PostMapping("/user")
    public Result add(@RequestBody User user){}
    
    // 3. 修改用户（改）
    @PutMapping("/user/{id}")
    public Result update(@PathVariable Integer id, @RequestBody User user){}
    
    // 4. 删除用户（删）
    @DeleteMapping("/user/{id}")
    public Result delete(@PathVariable Integer id){}
    
    // 5. 条件分页查询（普通参数）
    @GetMapping("/user/page")
    public Result page(@RequestParam Integer page, @RequestParam Integer size){}
```
# 六、混合使用场景
实际开发中可组合使用：
```java
// 路径ID + JSON请求体 同时接收
@PutMapping("/user/{id}")
public Result update(
@PathVariable Integer id,
@RequestBody User user
){}
```
# 七、常见报错总结
@PathVariable 不传参数：404 路径不存在
@RequestParam 必传参数为空：400 参数缺失
@RequestBody 未传 JSON / 格式错误：400 解析异常
GET 请求使用 @RequestBody：不推荐，部分服务器不支持
