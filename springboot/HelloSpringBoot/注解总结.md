## 三、@RestController
### 作用
`@RestController` 用来标记一个类是 Web 接口类。

### 示例
```java
@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Hello SpringBoot";
    }
}
```

### 理解
- 浏览器或接口工具访问对应路径时，会执行 Controller 中的方法，并把返回值直接返回给客户端。
- `@RestController` 常用于前后端分离项目中返回 JSON 数据。

---

## 四、@Autowired
### 作用
`@Autowired` 用来让 Spring 自动注入对象。

### 示例
```java
@Autowired
private JdbcTemplate jdbcTemplate;
```

### 理解
- 以前需要自己 new 对象：`StudentDao studentDao = new StudentDaoImpl();`
- 使用 Spring 后，对象由 Spring 容器管理，需要用的时候通过 `@Autowired` 注入。

**核心理解**：不用自己 new，让 Spring 帮我创建和注入对象。

---

## 五、@Component
### 作用
`@Component` 用来把普通类交给 Spring 管理。

### 示例
```java
@Component
public class MyComponent {
}
```

### 理解
被 `@Component` 标记的类，会被 Spring 扫描到，并创建成对象放入 Spring 容器。
之后可以通过 `@Autowired` 注入使用。

## 六、常见组件注解关系
这些注解本质上都属于组件注解：
```
@Component
├── @Controller
├── @RestController
├── @Service
└── @Repository
```

区别主要是语义不同：

| 注解 | 常用位置 |
| ---- | -------- |
| @RestController | 控制层，写接口 |
| @Service | 业务层 |
| @Repository | DAO / 数据访问层 |
| @Component | 普通组件类 |

---

## 七、常用请求注解
### 1. @GetMapping
用于查询：
```java
@GetMapping("/students")
public List<Student> findAll() {
    return students;
}
```

### 2. @PostMapping
用于新增：
```java
@PostMapping("/students")
public String add(@RequestBody Student student) {
    return "新增成功";
}
```

### 3. @PutMapping
用于修改：
```java
@PutMapping("/students/{id}")
public String update(@PathVariable Integer id, @RequestBody Student student) {
    return "修改成功";
}
```

### 4. @DeleteMapping
用于删除：
```java
@DeleteMapping("/students/{id}")
public String delete(@PathVariable Integer id) {
    return "删除成功";
}
```

## 八、参数相关注解
### 1. @PathVariable
从 URL 路径中获取参数。

**示例**：
```java
@GetMapping("/students/{id}")
public Student findById(@PathVariable Integer id) {
    return student;
}
```
访问：`/students/1`，id 的值就是 1。

### 2. @RequestBody
把请求体中的 JSON 转成 Java 对象。

**示例**：
```java
@PostMapping("/students")
public String add(@RequestBody Student student) {
    return "新增成功";
}
```
请求体：
```json
{
  "name": "张三",
  "age": 20
}
```
SpringBoot 会自动封装成 Student 对象。

## 九、Spring 与 SpringBoot 开发体验对比
### 传统 Spring
特点：
- 配置较多
- 需要手动配置很多组件
- 项目启动和整合成本较高

例如以前可能需要配置：
- SpringMVC
- DispatcherServlet
- 数据源
- JSON 转换
- Tomcat 部署

### SpringBoot
特点：
- 约定大于配置
- 自动配置
- 内置 Tomcat
- 通过 starter 快速引入功能

例如只要引入：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```
就可以快速开发 Web 接口。

## 十、SpringBoot 开发体验总结
SpringBoot 相比传统 Spring：
- 配置更少
- 启动更快
- 上手更简单
- 更适合快速开发 Web 项目
